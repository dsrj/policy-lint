🎯 Final Solution (clean + production way)

You need to:

✅ 1. Publish your provider binary (once)

Use GitHub Releases (best option)

Example:

https://github.com/YOUR_ORG/fw-analyzer/releases

Upload:

terraform-provider-fw-analyzer_v1.0.0_linux_amd64.zip

Inside zip:

terraform-provider-fw-analyzer


⸻

🔥 2. Use Terraform provider installation override (NOT dev_override)

In GitHub Actions, create .terraformrc:

cat <<EOF > $HOME/.terraformrc
provider_installation {
  filesystem_mirror {
    path    = "$HOME/.terraform.d/plugins"
    include = ["local/fw-analyzer"]
  }
  direct {}
}
EOF


⸻

🔥 3. Download provider into correct Terraform path

Terraform expects EXACT structure:

$HOME/.terraform.d/plugins/local/fw-analyzer/1.0.0/linux_amd64/


⸻

✅ GitHub Action (FINAL WORKING)

name: Firewall Policy Check

on: [pull_request]

jobs:
  analyze:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      # 🔥 Download prebuilt provider (NO BUILD)
      - name: Install fw-analyzer provider
        run: |
          mkdir -p ~/.terraform.d/plugins/local/fw-analyzer/1.0.0/linux_amd64

          curl -L \
          https://github.com/YOUR_ORG/fw-analyzer/releases/download/v1.0.0/terraform-provider-fw-analyzer \
          -o ~/.terraform.d/plugins/local/fw-analyzer/1.0.0/linux_amd64/terraform-provider-fw-analyzer

          chmod +x ~/.terraform.d/plugins/local/fw-analyzer/1.0.0/linux_amd64/terraform-provider-fw-analyzer

      # 🔥 Setup Terraform
      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_wrapper: false

      # 🔥 Terraform init (NO dev_override)
      - name: Terraform Init
        run: terraform init

      - name: Terraform Plan
        run: terraform plan -no-color


⸻

🔥 4. Terraform configuration (clean)

terraform {
  required_providers {
    fw-analyzer = {
      source  = "local/fw-analyzer"
      version = "1.0.0"
    }
  }
}
